# 用户命令

所有 QQ 用户都可以使用的命令。

## 通用命令

### `#qtine` - 查看机器人状态

查看 Qtine 的运行状态、版本、QQ 账号、运行时长。

```
#qtine
```

示例输出：

```
Qtine: [Running]
QQ: 123456789
Version: 1.0.0
Device: Windows
Uptime: 0d 1h 23m 45s
```

### `#help` / `#帮助` - 查看帮助

列出所有可用命令。

```
#help
#帮助
```

## 工具命令

### `/echo` / `/复读` - 复读消息

复读用户发送的内容（测试用）。

```
/echo 你好
/复读 你好
```

示例输出：

```
你好
```

::: tip
`/echo` 用于验证机器人是否正常工作。如果发了 `/echo` 没回复，说明消息链路有问题。
:::

## 命令前缀

| 前缀 | 用途 | 示例 |
|------|------|------|
| `#` | 通用命令 | `#qtine`、`#help` |
| `/` | 工具命令 | `/echo`、`/ban` |

## 使用示例

### 在群里使用

```
群成员 A: #qtine
机器人: Qtine: [Running] ...
群成员 A: #help
机器人: 可用命令列表...
群成员 A: /echo hello
机器人: hello
```

### 私聊使用

所有命令在私聊中同样可用，直接发送即可。

## 命令未匹配时

如果发送的消息不匹配任何命令：

- **群聊**：默认不回复（避免打扰）
- **私聊**：默认不回复
- **复读检测**：群聊中如果有人连发 3 条相同消息，机器人会自动 +1

可以通过 `repeat` 插件配置触发阈值。

## 注意事项

1. **命令区分大小写**：`#Help` 不会触发 `#help`
2. **前缀不能有空格**：`# help` 不会触发命令
3. **群聊需要 @机器人吗**：不需要，直接发即可（除非插件特别要求）
4. **频率限制**：短时间内发送过多命令会被限流，参考 [频率限制](/guide/rate-limit)
